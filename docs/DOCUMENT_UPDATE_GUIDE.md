### 2. docs/USAGE.md - 使用指南

**需要更新的内容**：

#### A. 新增章节：API费用节约

在"工作流程"章节后添加：

```markdown
## 💰 API费用节约机制

本系统通过感知哈希特征匹配智能节约API调用费用。

### 工作原理

1. 计算新图片的特征哈希（pHash/dHash/aHash）
2. 查询数据库中已有的违规图片
3. 计算汉明距离判断相似度
4. 根据距离决定是否调用IMS：
   - 距离 ≤ 3: 直接标记违规（节省100% API费用）
   - 距离 > 3: 调用IMS检测（保证准确性）

### 配置调整

如需调整相似度阈值，编辑 `scanner.py` 第197行：

```python
if distance <= 3:  # 可调整为2或4
    # 直接标记违规
```

**建议值**：
- 距离2: 更严格，准确性高，节约较少
- 距离3: 平衡点（默认推荐）
- 距离4: 更宽松，节约更多，可能误判

### 查看节约效果

扫描日志中会显示：

```
统计信息 - 总数: 100, 已扫描: 95, 违规: 3, 
跳过: 5, 错误: 0, 节约API: 25次
```

### 最大化节约建议

1. **建立违规样本库**
   - 首次扫描时积累违规图片
   - 后续扫描自动利用特征匹配
   
2. **定期维护**
   - 清理误判图片
   - 更新违规样本
   
3. **监控统计**
   - 关注"节约API"指标
   - 评估成本节省效果
```
